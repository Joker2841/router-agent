The problem

Cheaper AI without giving up quality

Track 1 ranks agents by how few Fireworks tokens they spend, as long as they stay above the accuracy gate on nineteen hidden tasks across eight categories. Local inference counts as zero tokens. So the winning move is simple to say and hard to do: answer everything you can on your own, and pay for a hosted model only when you truly have no other choice.

Our approach

Answer locally, prove the answer, escalate only if forced

  • A small language model runs inside the container on CPU, so every answer it gives is free.
  • We fine-tuned that model on our own generated data so it matches the exact format each category is graded on.
  • Math and logic are turned into short Python programs that we actually run, so the answer is computed and checked, not guessed.
  • A hosted model is called only when a task somehow returns nothing. In our testing that never happened.
How it works

One pipeline across eight task categories

Computed and verified

Math is written as a program and executed. Logic puzzles are solved by enumerating every possibility in code, with prose reasoning as a backup. Generated and debugged code is run against tests before we trust it.

Answered by the model

Sentiment, summarization, and factual questions are handled directly by the fine-tuned model, formatted to the grader's rules.

Named entities

A dedicated spaCy model extracts people, organizations, locations, and dates. It is deterministic, reliable, and free.

The engineering fights

What almost stopped us, and how we fixed it

  • Bigger 3B and 4B models were more accurate but timed out on the two core judging box. We moved to a fine-tuned 1.5B that is both fast and accurate enough.
  • Math kept returning blank. The model wrote notebook-style code that ended on a bare value and never printed it. We taught the runner to print that final value, and math went from zero to perfect.
  • Multi-part questions lost half the answer. A sharper instruction made the model print every quantity it was asked for.
  • Logic wrote buggy constraint code. Adding a prose reasoning fallback turned a weak spot into a solved category.
